# javaWRT (OpenJDK 8 для OpenWrt)

**javaWRT** — это кастомный установочный пакет в формате `.apk` для развертывания среды выполнения Java (OpenJDK 8) на роутерах под управлением OpenWrt 25.12+ (архитектура x86_64).

Особенность этого пакета в том, что он обходит жесткий сканер зависимостей OpenWrt (ELF-анализатор). Внутренние файлы Java упаковываются в payload-архив на этапе сборки и автоматически разворачиваются в системе при установке пакета.

---

## 📦 Установка

Поскольку сборки Java часто используют стандартные Linux-библиотеки, перед запуском может потребоваться слой совместимости `gcompat` (так как OpenWrt использует легковесную `musl libc`). Убедитесь, что он установлен в вашей системе.

### Основной способ: через веб-интерфейс (LuCI)

Самый простой способ установки — загрузить пакет напрямую через графический интерфейс роутера:

1. Перейдите во вкладку **Releases** в этом репозитории и скачайте актуальный файл `javaWRT_..._x86_64.apk` на свой компьютер.
2. Откройте веб-интерфейс вашего роутера (LuCI).
3. Перейдите в раздел **Система -> Программное обеспечение** (System -> Software).
4. Нажмите кнопку **Upload Package...** (Загрузить пакет...) и выберите скачанный `.apk` файл.
5. Нажмите **Install** (Установить).
6. Для применения системных переменных (чтобы команда `java` работала из любого места) переподключитесь к вашей SSH-сессии или перезагрузите терминал.

### Альтернативный способ: через терминал (SSH)

Если вы предпочитаете командную строку или хотите установить пакет из подключенного репозитория:

1. Подключитесь к роутеру по SSH.
2. Обновите списки и установите пакет совместимости:
   ```bash
   apk update
   apk add gcompat
   ```
3. Скачайте и установите пакет javaWRT (при ручной установке используйте флаг доверия):
   ```bash
   apk add --allow-untrusted /tmp/javaWRT.apk
   ```
4. Проверьте работоспособность:
  ```bash
  java -version
```
## 👥 Авторство и история проекта
Проект создан благодаря совместным усилиям:

- Ball-bess — Сопровождающий / Упаковщик. Адаптация проекта под среду OpenWrt SDK, создание Makefile и упаковка в формат .apk с использованием payload-архива для обхода сборочных ограничений.
**Оригинальный скрипт от Niwas (историческая справка):**
Изначально проект базировался на классическом подходе ручной распаковки бинарников из .tar.gz архива
```bash
mkdir -p /usr/lib/jvm
tar -xzf openjdk-8_x86_64.tar.gz
'export PATH="/usr/lib/jvm/bin:$PATH"' >> /etc/profile
source /etc/profile
```
В текущей реализации javaWRT логика этого скрипта глубоко интегрирована в секции postinst и prerm сборочного Makefile, что позволяет пакетному менеджеру OpenWrt корректно устанавливать и бесследно удалять файлы.
